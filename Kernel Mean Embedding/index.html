KERNEL MEAN EMBEDDING

Welcome to my first-ever blog post! No better way to start off than with an attempt to explain Kernel Mean Embedding. This post should be accessible to as many people from different sorts of backgrounds as possible. Don't let the theory scare you as this will become much clearer with the toy examples. These examples are written in Julia, which many of you won't be familiar with. However, Julia's code is very easy to interpret! The outline of the theory is based on the review article of Kernel Mean Embedding by Muandet et al. Finally, the Pluto notebooks can be found on my Github and all of my sources are referenced at the bottom of this page.

Kernel Methods

Consider some data which can be evaluated by the classical machine learning algorithms perceptron, support vector machine and principal component analysis. These algorithms consider the data, x,x′∈Xx,x' \in \mathcal Xx,x′∈X, with X\mathcal XX a non empty set, through their inner product ⟨x,x′⟩\langle x, x' \rangle⟨x,x′⟩. Another way to view this inner product is as a linear similarity measure between the elements of X\mathcal XX. But what if the data is more complex and cannot be linearly evaluated?

Let's consider this arbitrary classification problem, in which two data circles cannot be linearly separated.

Here the kernel methods come into play! Kernel methods replace the inner product ⟨x,x′⟩\langle x, x' \rangle⟨x,x′⟩ as a similarity measure with a non-linear one. Consider the non-linear transformation:

ϕ:X→Fx↦ϕ(x) \phi:\mathcal X\to \mathcal F \\ x \mapsto \phi (x) ϕ:X→Fx↦ϕ(x)

from X\mathcal XX to the high-dimensional feature space F\mathcal FF. In this new feature space the inner product can be evaluated:

k(x,x′)=⟨ϕ(x),ϕ(x′)⟩F, k(x,x') = \langle \phi(x), \phi(x') \rangle_{\mathcal{F}}, k(x,x′)=⟨ϕ(x),ϕ(x′)⟩F​,

where ⟨⋅,⋅⟩F\langle \cdot, \cdot \rangle_{\mathcal{F}}⟨⋅,⋅⟩F​ is the inner product of F\mathcal FF, ϕ\phiϕ is the feature map, and kkk is the kernel function which defines a non-linear similarity measure between xxx and x′x'x′. The above mentioned algorithms can then be used in this new space, by swapping out ⟨x,x′⟩\langle x, x' \rangle⟨x,x′⟩ with ⟨ϕ(x),ϕ(x′)⟩F\langle \phi(x), \phi(x') \rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​. So, the algorithm itself does not change, rather the space in which it operates!

To illustrate this, let's perform the following feature mapping of the concentric circles: ϕ:(x1,x2)↦(x12,2x1x2,x22)\phi:(x_{1}, x_{2})\mapsto ({x_{1}}^2,\sqrt{2}x_{1}x_{2}, {x_{2}}^2)ϕ:(x1​,x2​)↦(x1​2,2​x1​x2​,x2​2). In this new higher dimensional feature space we can find a linear model that defines a decision function which separates the two circles easily. We can calculate the kernel function, for this example:

⟨ϕ(x),ϕ(x′)⟩R3=x12x1′2+2x1x2x1′x2′+x22x2′2=(x1x1′+x2x2′)2=⟨x,x′⟩R22. \langle \phi(x), \phi(x') \rangle_{\mathbb{R_{3}}} = {x_{1}}^2{x_{1}'}^2 + 2x_{1}x_{2}x_{1}'x_{2}' + {x_{2}}^2{x_{2}'}^2\\ = {(x_{1}x_{1}' + x_{2}x_{2}')}^2\\ = {\langle x, x' \rangle_{\mathbb{R_{2}}}}^2. ⟨ϕ(x),ϕ(x′)⟩R3​​=x1​2x1′​2+2x1​x2​x1′​x2′​+x2​2x2′​2=(x1​x1′​+x2​x2′​)2=⟨x,x′⟩R2​​2.

So our new similarity measure is the square of the inner product in X\mathcal XX.

So the core idea of kernel methods is taking data which lives in an input space where it's not easy to perform machine learning and transform this data to a higher dimensional space where linear models can be used.

As seen above in the example, to evaluate equation 2 we need to work in 2 steps: explicitly constructing the feature maps ϕ(x)\phi(x)ϕ(x) and then evaluating the inner product ⟨ϕ(x),ϕ(x′)⟩F\langle \phi(x), \phi(x') \rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​. This can become a problem when ϕ(x)\phi(x)ϕ(x) defines a transformation to a high-dimensional featue space. The kernel trick offers a solution to this by evaluating ⟨ϕ(x),ϕ(x′)⟩F\langle \phi(x), \phi(x') \rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​ without explicitly constructing the feature maps. For the example: we can just consider k(x,x′)=⟨x,x′⟩2k(x,x') = {\langle x, x' \rangle}^2k(x,x′)=⟨x,x′⟩2. Illustration of the kernel trick:

Now let's discuss an important requirement needed in order for the kernel trick to do its magic: kkk has to be positive definite. What does this mean? First we have to define the n×nn \times nn×n-Gram or kernel matrix Kij:=k(xi,xj)K_{ij} := k(x_{i},x_{j})Kij​:=k(xi​,xj​), the collection of all pairwise inner products within the set of data vectors xxx.

A symmetric function kkk is positive definite on X\mathcal XX if the Gram matrix is positive definite, that is,

∑i=1n∑j=1jcicjk(xi,xj)≥0,∀xi∈X \sum{i=1}^n\sum{j=1}^j c_{i}c_{j}k(x_{i},x_{j}) \geq 0, \forall x_{i} \in \mathcal X ∑i=1n∑j=1jci​cj​k(xi​,xj​)≥0,∀xi​∈X

Equation 4 holds for any n∈Nn \in \mathbb Nn∈N, all finite sequences of points x1,...,xnx_{1},...,x_{n}x1​,...,xn​ in X\mathcal XX and all choices of nnn real-valued coefficients c1,...,cn∈Rc_{1},...,c_{n} \in \mathbb Rc1​,...,cn​∈R. It's the positive definiteness of the kernel function that guarantees the existence of a dot product space F\mathcal FF and a feature map ϕ:X→F\phi:\mathcal X\to \mathcal Fϕ:X→F such that k(x,x′)=⟨ϕ(x),ϕ(x′)⟩Fk(x,x') = \langle \phi(x), \phi(x') \rangle_{\mathcal{F}}k(x,x′)=⟨ϕ(x),ϕ(x′)⟩F​ without the need for the explicit construction of ϕ\phiϕ.

Another important property of a positive definite kernel is that it induces a space of functions from X\mathcal XX to R\mathbb RR called a Reproducing Kernel Hilbert Space RKHS H\mathcal HH, which is why the p.d. kernel is also called a reproducing kernel.

Two important properties define an RKHS: First, for any x∈Xx \in \mathcal Xx∈X, the function k(x,⋅):y↦k(x,y)k(x,\cdot):y\mapsto k(x,y)k(x,⋅):y↦k(x,y) is an element of H\mathcal HH. So, when kkk is used, the feature space F\mathcal FF is the associated RKHS H\mathcal HH;

k:X→H⊂RXx↦k(x,⋅) k:\mathcal X \to \mathcal H \subset {\mathbb R}^{\mathcal X}\\ x \mapsto k(x,\cdot) k:X→H⊂RXx↦k(x,⋅)

Where RX\mathbb R^{\mathcal X}RX denotes the vector space of functions from X\mathcal XX to R\mathbb RR, we call kkk the canonical feature map.

Second, a function k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R is called a reproducing kernel of H\mathcal HH if k(⋅,x)∈Hk(\cdot,x) \in \mathcal Hk(⋅,x)∈H for all x∈Xx \in \mathcal Xx∈X and the reproducing property

f(x)=⟨f,k(⋅,x)⟩H f(x) = \langle f,k(\cdot,x) \rangle_{\mathcal H} f(x)=⟨f,k(⋅,x)⟩H​

holds for all f∈Hf \in \mathcal Hf∈H and all x∈Xx \in \mathcal Xx∈X. Important for us: k(x,x′)=⟨k(x,⋅),k(x′,⋅)⟩Hk(x,x')=\langle k(x,\cdot),k(x',\cdot) \rangle_{\mathcal H}k(x,x′)=⟨k(x,⋅),k(x′,⋅)⟩H​.

Aronszajn (1950): “There is a one-to-one correspondance between the reproducing kernel kkk and the RKHS H\mathcal HH”.

Kernel Mean Embedding of Marginal Distributions

So how can we extend this mapping to marginal distributions? We simply take the mapping µµµ which defines the representer in H\mathcal HH of any distribution P\mathbb PP. The distribution P\mathbb PP is transformed into an element, the mean embedding µP{µ}_{\mathbb P}µP​ in an RKHS matching the positive definite kernel kkk. The element is the expected value in the RKHS and since P\mathbb PP is a probability density distribution, it can be written as an integral!

µ:P→HP↦∫k(⋅,x) dPx µ: \mathcal P \to \mathcal H\\ \mathbb P \mapsto \int k(\cdot,x)\, \mathrm{d\mathbb P}x µ:P→HP↦∫k(⋅,x)dPx

Visual representation of this mean embedding:

Now how much information can this mean representation capture about the distribution P\mathbb PP? This depends on the used kernel! This can range from only the first moment of P\mathbb PP to all information of P\mathbb PP.

k(x,x′)=⟨x,x′⟩k(x,x')=\langle x,x' \ranglek(x,x′)=⟨x,x′⟩ : the first moment of P\mathbb PP k(x,x′)=(⟨x,x′⟩+1)pk(x,x')={(\langle x,x' \rangle+1)}^{p}k(x,x′)=(⟨x,x′⟩+1)p (polynomial kernels): moments of P\mathbb PP up to order p∈Np \in \mathbb Np∈N k(x,x′)k(x,x')k(x,x′) is universal/characteristic: all information of P\mathbb PP

A kernel kkk is characteristic if the map

P↦µP \mathbb P \mapsto µ_{\mathbb P} P↦µP​

is injective, i.e., ∥µP−µQ∥H=0{\|µ_{\mathbb P} - µ_{\mathbb Q} \|}_{\mathcal H} = 0∥µP​−µQ​∥H​=0 if and only if P=Q\mathbb P = \mathbb QP=Q. This injectivity of the map P↦µP\mathbb P \mapsto µ_{\mathbb P}P↦µP​ ensures that the RKHS embedding is suitibale for regression problems (each element in the feature space corresponds to one unique distribution in the original space)!

An example of a characteristic kernel, also the kernel used in all toy examples, is the Gaussian kernel, which is part of the class of kernels called radial basis functions (RBFs):

kRBF(x,x′)=exp(−∥x−x′∥22σ2) k^{RBF}(x,x') = exp(-\frac{\|x-x'\|^{2}}{2{\sigma}^{2}}) kRBF(x,x′)=exp(−2σ2∥x−x′∥2​)

with σ>0\sigma > 0σ>0 the bandwith parameter. The Gram matrix of the Gaussian kernel becomes a matrix of ones for σ→∞\sigma \to \inftyσ→∞ and an indentity matrix for σ→0\sigma \to 0σ→0. Which means for the former that all input is the same, and for the latter that all input is completely unique. This RBF kernel is a stationary kernel, which means that it can be described as a function of the difference of its inputs. The RBF kernel is also a universal kernel which means it can represent any smooth function with a high degree of accuracy , assuming chosen the right bandwith parameter. It must be noted that all universal kernels are characteristic, but characteristic kernels may not be universal.
