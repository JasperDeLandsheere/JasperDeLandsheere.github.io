Kernel methods

Introduction

Inner products serve as a powerful tool in many established machine learning algorithms, such as principal component analysis (PCA)[2] [3], perceptron [4] [5], and support vector machine (SVM) [6]. These algorithms consider the data, e.g., x,x′∈X\mathbf{x},\mathbf{x}' \in \mathcal Xx,x′∈X, with X\mathcal XX a non empty set, through their inner product ⟨x,x′⟩\langle\mathbf{x}, \mathbf{x}'\rangle⟨x,x′⟩, which can be interpreted as a similarity measure between x\mathbf{x}x and x′\mathbf{x}'x′. But real-life data is often complex and the class of linear functions induced by the inner products might prove to be insufficient. The aim of kernel methods is to handle complex data which can't be linearly evaluated, by replacing ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}'\rangle⟨x,x′⟩ with some other (non-linear) similarity measure.

Naturally, an extension of ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}'\rangle⟨x,x′⟩ can be made by explicitly applying a non-linear transformation:

ϕ:X→F,x↦ϕ(x), \begin{split} \phi:\mathcal{X} &\to \mathcal{F}, \\ x &\mapsto \phi(x), \end{split} ϕ:Xx​→F,↦ϕ(x),​

from X\mathcal XX to the high-dimensional feature space F\mathcal FF. In this new feature space the inner product can be evaluated:

k(x,x′):=⟨ϕ(x),ϕ(x′)⟩F, k(\mathbf{x},\mathbf{x}') := \langle\phi(\mathbf{x}), \phi(\mathbf{x}')\rangle_{\mathcal{F}}, k(x,x′):=⟨ϕ(x),ϕ(x′)⟩F​,

where ⟨⋅,⋅⟩F\langle\cdot, \cdot \rangle_{\mathcal{F}}⟨⋅,⋅⟩F​ is the inner product of F\mathcal FF and ϕ\phiϕ is called the feature map. kkk is the kernel function which defines a non-linear similarity measure between x\mathbf{x}x and x′\mathbf{x}'x′. By substituting ⟨x,x′⟩\langle\mathbf{x}, \mathbf{x}'\rangle⟨x,x′⟩ with ⟨ϕ(x),ϕ(x′)⟩F\langle\phi(\mathbf{x}), \phi(\mathbf{x}')\rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​ a non-linear extension of the algorithms that consider data through ⟨x,x′⟩\langle\mathbf{x}, \mathbf{x}'\rangle⟨x,x′⟩, can be made. So, a linear algorithm in F\mathcal FF corresponds to a non-linear counterpart in the original input space. It is important to note that the algorithm stays the same, only the space in which the algorithm operates, changes.

Consider a polynomial feature mapping on the data points ϕ:(x1,x2)↦(x12,2x1x2,x22)\phi:(x_{1}, x_{2})\mapsto ({x_{1}}^2,\sqrt{2}x_{1}x_{2}, {x_{2}}^2)ϕ:(x1​,x2​)↦(x1​2,2​x1​x2​,x2​2). The inner product of F\mathcal{F}F can be calculated:

So, the new similarity measure is the square of the inner product in X\mathcal XX. When applied on the data points, one can obtain the following result illustrated in the figure below.

In this new higher dimensional feature space, R3\mathbb{R}^3R3, one can find an appropriate learning algorithm that defines a decision function which can separate the two circles easily. So, in the original input space the underlying function defining the interactions between all the features of data might not be clear. That's why the data is transformed to higher dimensional feature space where the underlying function might become more clear. In this example, by mapping the data points to a higher dimensional feature space, one can distinguish a cone shape in the transformed data. The core idea of kernel methods is taking data which lives in an input space where it's not easy to perform machine learning and transform this data to a higher dimensional space where effective use of learning algorithms can be made.

The kernel trick

As seen in the above example, to evaluate equation 2 one needs to work in two steps: i) explicitly construct the feature maps ϕ(x)\phi(\mathbf{x})ϕ(x), and ii) subsequently evaluate the inner product ⟨ϕ(x),ϕ(x′)⟩F\langle\phi(\mathbf{x}), \phi(\mathbf{x}')\rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​. This can become a problem when ϕ(x)\phi(\mathbf{x})ϕ(x) defines a computationally expensive transformation to a high-dimensional feature space. Fortunately, there exists a solution to this by evaluating ⟨ϕ(x),ϕ(x′)⟩F\langle\phi(\mathbf{x}), \phi(\mathbf{x}')\rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​ without explicitly constructing the feature maps. This is a core idea of kernel methods and is often called "the kernel trick" in the machine learning community[1].

A visual representation of the kernel trick is illustrated in the figure below. For the above mentioned example one can just consider k(x,x′)=⟨x,x′⟩2k(\mathbf{x},\mathbf{x}') = {\langle\mathbf{x}, \mathbf{x}'\rangle}^2k(x,x′)=⟨x,x′⟩2, rather than calculating the feature maps explicitly. In other words, to avoid using the coordinates of the vectors in the new feature space, a similarity measure is used in that space, which can be put in an algorithm that only needs the value of this measure.

What are the requirements for the kernel trick to do its magic? If kkk is positive definite there always exists a feature map ϕ:X→F\phi : \mathcal X \to \mathcal Fϕ:X→F such that k(x,x′)=⟨ϕ(x),ϕ(x′)⟩Fk(\mathbf{x},\mathbf{x}') = \langle\phi(\mathbf{x}), \phi(\mathbf{x}')\rangle_{\mathcal{F}}k(x,x′)=⟨ϕ(x),ϕ(x′)⟩F​ [7] without the explicit construction of ϕ\phiϕ [6] [8]. So, the kernel function is used as a way to calculate an inner product ⟨ϕ(x),ϕ(x′)⟩\langle\phi(\mathbf{x}), \phi(\mathbf{x}')\rangle⟨ϕ(x),ϕ(x′)⟩ in a high-dimensional feature space H\mathcal HH for some data points x,x′∈X\mathbf{x},\mathbf{x}' \in \mathcal Xx,x′∈X. The collection of all these pairwise inner products within the set of data vectors x\mathbf{x}x is defined as the N×NN \times NN×N-Gram or kernel matrix Kij:=k(xi,xj)K_{ij} := k(\mathbf{x}_{i},\mathbf{x}_{j})Kij​:=k(xi​,xj​). The comparison function kkk is a positive definite kernel on X\mathcal XX if it is symmetric, i.e., k(x,x′)=k(x′,x)k(\mathbf{x}, \mathbf{x}') = k(\mathbf{x}',\mathbf{x})k(x,x′)=k(x′,x), and the Gram matrix is positive definite:

∑i=1N∑j=1Ncicjk(xi,xj)≥0, \displaystyle\sum_{i=1}^{N}\displaystyle\sum_{j=1}^{N} c_{i}c_{j}k(\mathbf{x}_{i},\mathbf{x}_{j}) \geq 0, i=1∑N​j=1∑N​ci​cj​k(xi​,xj​)≥0,

for any N∈NN \in \mathbb NN∈N, all finite sequences of points (x1,...,xN)∈XN(x_{1},...,x_{N}) \in \mathcal X^{N}(x1​,...,xN​)∈XN and any NNN real-valued coefficients (c1,...,cN)∈RN(c_{1},...,c_{N}) \in \mathbb R^{N}(c1​,...,cN​)∈RN [9].

Recurrent kernel Hilbert space

Another important property of a positive definite kernel is that it induces a space of functions from X\mathcal XX to R\mathbb RR called a reproducing kernel Hilbert space (RKHS) H\mathcal HH, which is why the kernel is also called a reproducing kernel [7]. It's important to note that the RKHS is a space of functions from X\mathcal XX to R\mathbb RR. In other words, each data point x\mathbf{x}x in X\mathcal XX will be represented by a function ϕ(x)\phi(\mathbf{x})ϕ(x) in H\mathcal HH.

An RKHS has two properties important to kernel mean embedding: (i) for any x∈X\mathbf{x} \in \mathcal Xx∈X, the function k(x,⋅):y↦k(x,y)k(\mathbf{x},\cdot):\mathbf{y} \mapsto k(\mathbf{x},\mathbf{y})k(x,⋅):y↦k(x,y) is an element of H\mathcal HH. So, whenever the kernel kkk is used, the feature space F\mathcal FF is the RKHS H\mathcal HH associated with this kernel. This can be considered as the canonical feature map:

k:X→H⊂RX,x↦k(x,⋅), \begin{split} k:\mathcal X &\to \mathcal H \subset {\mathbb R}^{\mathcal X}, \\ x &\mapsto k(\mathbf{x},\cdot), \end{split} k:Xx​→H⊂RX,↦k(x,⋅),​

where RX\mathbb R^{\mathcal X}RX denotes the vector space of functions from X\mathcal XX to R\mathbb RR; (ii) a function k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R is called a reproducing kernel of H\mathcal HH if k(⋅,x)∈Hk(\cdot,\mathbf{x}) \in \mathcal Hk(⋅,x)∈H for all x∈X\mathbf{x} \in \mathcal Xx∈X and the reproducing property

f(x)=⟨f,k(⋅,x)⟩H f(\mathbf{x}) = \langle f,k(\cdot,\mathbf{x})\rangle_{\mathcal H} f(x)=⟨f,k(⋅,x)⟩H​

holds for all f∈Hf \in \mathcal Hf∈H and all x∈X\mathbf{x} \in \mathcal Xx∈X. In particular: if f(x′=k(x′,⋅))f(\mathbf{x'} = k(\mathbf{x}',\cdot))f(x′=k(x′,⋅)) for some x′∈X\mathbf{x}' \in \mathcal Xx′∈X, k(x,x′)=⟨k(x,⋅),k(x′,⋅)⟩Hk(\mathbf{x},\mathbf{x}')=\langle k(\mathbf{x},\cdot),k(\mathbf{x}',\cdot)\rangle_{\mathcal H}k(x,x′)=⟨k(x,⋅),k(x′,⋅)⟩H​.

Aronszajn (1950): “There is a one-to-one correspondence between the reproducing kernel kkk and the RKHS H\mathcal HH”.

Kernel functions

The kernel trick not only delivers powerful (non-linear) learning algorithms, but also paves the path for domain experts to invent certain kernels which are suitable for specific applications. The kernel trick does not only apply to Euclidean data, but also to non-Euclidean structured data, functional data, and other domains on which a positive definite kernel may be applied [8] [10]. Various kernels have been proposed in various application domains [11] and for different types of data, such as strings, graphs and trees [8] [10] [12].

The kernel used in this thesis is called the Gaussian kernel, which is a member of the class of kernels called radial basis functions (RBFs):

kRBF(x,x′)=exp(−∥x−x′∥22σ2), k^{RBF}(\mathbf{x},\mathbf{x}') = exp(-\frac{\|\mathbf{x}-\mathbf{x}'\|^{2}}{2{\sigma}^{2}}), kRBF(x,x′)=exp(−2σ2∥x−x′∥2​),

with σ>0\sigma > 0σ>0 the bandwidth parameter. The Gram matrix of the Gaussian kernel becomes a matrix of ones for σ→∞\sigma \to \inftyσ→∞ and an identity matrix for σ→0\sigma \to 0σ→0. Which means for σ→∞\sigma \to \inftyσ→∞ that all instances are the same, and for the σ→0\sigma \to 0σ→0 that all instances are completely unique, making it a relevant interpretation as a similarity measure [13]. RBF kernels are stationary kernels, which means that they can be described as functions of the differences of their input. RBF kernels are also universal kernels, which means they can represent any smooth function with a high degree of accuracy, assuming being able to find the right bandwidth parameter [11] [14].

For additional information on the properties of (reproducing kernel) Hilbert spaces and the important theorems of Mercer and Bochner, the reader is advised to read Muandet et al. [1], Mercer [9], and Bochner [15], respectively. For examples of learning algorithms that use the implicit representation of data points in kernel methods, such as support vector machine (SVM), gaussian process (GP), and neural tangent kernel (NTK), the reader is referred to read Steinwart & Christmann [16], Rasmussen [17], and Jacot et al. [18], respectively.

Kernel mean embedding of marginal distributions

From data points to probability measures

Having reviewed the above section on kernel methods one could wonder the practicality of extending kernel methods from individual data points to probability distributions. In many real-life learning problems, however, it could be argued that it is more appropriate to represent the training data as probability distributions rather than individual data points.

For example, in many situations data is missing or uncertain. As a specific example, gene expression data originating from microarray experiments are known to be very noisy, due to different sources of variabilities [19]. To battle this, each array can be represented as a probability distribution. Another reason for the preference of probability distributions can be computational challenges when dealing with large amounts of training data [20].

Let k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R be a real-valued positive definite kernel associated with the Hilbert space H\mathcal HH, with X\mathcal XX a non-empty set. The reproducing property lets the kernel evaluation be interpreted as an inner product in H\mathcal HH induced by a map from X\mathcal XX into H\mathcal HH

x↦k(x,⋅). \mathbf{x} \mapsto k(\mathbf{x,\cdot}). x↦k(x,⋅).

Basically, k(x,⋅)k(\mathbf{x,\cdot})k(x,⋅) is a high-dimensional representer of x\mathbf{x}x and because of the reproducing property k(x,⋅)k(\mathbf{x,\cdot})k(x,⋅) it is also a representer of evaluation of any function in H\mathcal HH on the data point x\mathbf{x}x. This lets feature map ϕ\phiϕ to be extended to the space of probability distributions through the mapping of μ\muμ which defines the representer in H\mathcal HH of any distribution P\mathbb PP [20] :

μ:M+1(X)→HP↦∫Xk(x,⋅)dP(x), \begin{split} \mu: M_+^{1}(\mathcal X) &\to \mathcal H\\ \mathbb P &\mapsto \int_{\mathcal X} k(\mathbf{x},\cdot) \mathrm{d\mathbb P}(\mathbf{x}), \end{split} μ:M+1​(X)P​→H↦∫X​k(x,⋅)dP(x),​

with MX1(X)M_X^{1}(\mathcal X)MX1​(X) the space of probability measures over a measurable space X\mathcal XX. The distribution P\mathbb PP is transformed into an element, the mean embedding μP{\mu}_{\mathbb P}μP​, in an RKHS corresponding to the positive definite kernel kkk, hence the name kernel mean embedding:

ϕ(P)=μP:=EX∼P[k(X,⋅)]=∫Xk(x,⋅)dP(x). \phi(\mathbb P) = \mu_{\mathbb P} := \mathbb E_{X\sim \mathbb P}[k(X,\cdot)] = \int_{\mathcal X} k(\mathbf{x},\cdot) \mathrm{d\mathbb P}(\mathbf{x}). ϕ(P)=μP​:=EX∼P​[k(X,⋅)]=∫X​k(x,⋅)dP(x).

The element is the expected value in the RKHS and since P\mathbb PP is a probability density distribution, it can be interpreted as an integral [21] [22]. In other words, in equation 10 the distribution P\mathbb{P}P is transformed into an element in the RKHS H\mathcal{H}H corresponding to the kernel kkk, therefore RKHS methods can be extended to probability measures. Kernel mean representation

One can wonder how much information about the distribution P\mathbb PP this kernel mean embedding can capture. This depends on the used kernel and can range from only the first moment of P\mathbb PP to all information of P\mathbb PP. Consider following examples with their corresponding captured information:

  • k(x,x′)=⟨x,x′⟩k(\mathbf{x},\mathbf{x}')=\langle\mathbf{x},\mathbf{x}'\ranglek(x,x′)=⟨x,x′⟩ : the first moment of P\mathbb PP

  • k(x,x′)=(⟨x,x′⟩+1)pk(\mathbf{x},\mathbf{x}')={(\langle\mathbf{x},\mathbf{x}'\rangle+1)}^{p}k(x,x′)=(⟨x,x′⟩+1)p: moments of P\mathbb PP up to order p∈Np \in \mathbb Np∈N

  • k(x,x′)k(\mathbf{x},\mathbf{x}')k(x,x′) is universal/characteristic: all information of P\mathbb PP

The first and second example are called the linear and polynomial kernel, respectively. The linear kernel, which equals the computation of the inner product, lets μP\mu_{\mathbb P}μP​ retain the first moment of P\mathbb PP. For the polynomial kernel of order ppp, the mean map equals up to the ppp-th moment of P\mathbb PP. For some other explicit examples the reader is referred to Smola et al. [21], Fukumizu et al. [23], Sriperumbudur et al. [24], Gretton et al. [25], and Schölkopf et al. [26].

In the third example, characteristic and universal kernels are mentioned. Characteristic kernels are a class of kernel functions for which the kernel mean embedding captures all information about the distribution P\mathbb PP. A kernel kkk is characteristic if the map P↦μP\mathbb P \mapsto \mu_{\mathbb P}P↦μP​ is injective, i.e., ∥μP−μQ∥H=0{\|\mu_{\mathbb P} - \mu_{\mathbb Q} \|}_{\mathcal H} = 0∥μP​−μQ​∥H​=0 if and only if P=Q\mathbb P = \mathbb QP=Q [27]. The Gaussian kernel used in this thesis is a characteristic kernel [23]. It must be noted that all universal kernels are characteristic, but characteristic kernels may not be universal [25] [28].

This injectivity of the map P↦μP\mathbb P \mapsto \mu_{\mathbb P}P↦μP​ ensures that the RKHS embedding is suitable for regression problems, since each element in the feature space corresponds to one unique distribution in the original space. In other words, no information is lost when mapping the distribution into the RKHS.

Empirical estimate of mean embeddings

Toy problem 1: kernel PCA

Principal component analysis (PCA) is a classical linear technique for dimensionality reduction, data exploration, feature extraction and data visualization. The general outline of PCA is performing a linear projection of the data onto a lower dimensional subspace such that the variance of the projected data is maximized. Kernel PCA is simply a non-linear extension of PCA by executing these principles in the Hilbert space and making use of the kernel trick.

Then the kernel matrix is generated over the x and y matrices, each row is taken as one element, hence the use of RowVecs(). Now everything is obtained to solve equation 21. Finally, the eigenvector problem is solved. Kernel PCA was used on the input data, using four different values for the scale factor. The results were plotted in the figure below, only the first two principal components are shown.

function kernelPCA(X, Y, scale_factor)
	# scale_factor = inverse of lengthscale, X = matrix x1 points, Y = matrix x2 points, each row of the points matrices is an input element
	# Defining the Gaussian kernel
	k = SqExponentialKernel() ∘ ScaleTransform(scale_factor)
	# Generating the kernel over the x1 and x2 points
	K = kernelmatrix(k, RowVecs(X), RowVecs(Y))
	# Calculating the centralized kernel matrix
	K_centered = K .- mean(K, dims =1) .- mean(K, dims =2) .+ mean(K)
	# Solving the eigenvector problem
	eigenval, eigenvect = eigen(K_centered, sortby=$\lambda$ -> -real($\lambda$))
	eigenvect = real(eigenvect[:, 1:2])
	eigenval = real(eigenval[1:2])
	return eigenvect,eigenval,K
end

In the next toy problem kernel mean embedding is used to define a metric for probability functions, called the maximum mean discrepancy (MMD). This metric is very important for solving problems in statistics and machine learning when handling distributions.

Toy problem 2: maximum mean discrepancy

Consider following arbitrary toy problem, illustrated in the figure below. Given noisy data points which lay in a circular shape, is it possible to find a model fit of 100 equally spaced points which lay on a circle with a certain radius that represents the input data?

This problem can be solved using kernel mean embedding, particularly using the maximum mean discrepancy (MMD). The maximum mean discrepancy corresponds to the RKHS distance between mean embeddings [25] [32] :

MMD2(P,Q,H)=∥μP−μQ∥H2=∥μP∥H−2⟨μP,μQ⟩H+∥μQ∥H. \begin{split} MMD^{2}(\mathbb P, \mathbb Q, \mathcal H) &= {\|\mu_{\mathbb P} - \mu_{\mathbb Q}\|}_{\mathcal H}^{2} \\ &= {\|\mu_{\mathbb P}\|}_{\mathcal H} -2\langle{\mu_{\mathbb P}, \mu_{\mathbb Q}}\rangle_{\mathcal H} + {\|\mu_{\mathbb Q}\|}_{\mathcal H}. \end{split} MMD2(P,Q,H)​=∥μP​−μQ​∥H2​=∥μP​∥H​−2⟨μP​,μQ​⟩H​+∥μQ​∥H​.​

Given {xi}i=1n∼P\{\mathbf{x}_i\}_{i=1}^n \sim \mathbb P{xi​}i=1n​∼P and {yj}j=1n∼Q\{\mathbf{y}_j\}_{j=1}^n \sim \mathbb Q{yj​}j=1n​∼Q, the empirical MMD is [32] :

MMD^u2(P,Q,H)=1n(n−1)∑i=1n∑j≠ink(xi,xj)−2nm∑i=1n∑j=1mk(xi,yj)+1m(m−1)∑i=1m∑j≠imk(yi,yj). \begin{split} \widehat{MMD}_u^{2}(\mathbb P, \mathbb Q, \mathcal H) &= \frac{1}{n(n-1)}\displaystyle\sum_{i=1}^{n}\displaystyle\sum_{j\neq i}^{n}k(\mathbf{x}_{i},\mathbf{x}_{j}) - \frac{2}{nm}\displaystyle\sum_{i=1}^{n}\displaystyle\sum_{j= 1}^{m}k(\mathbf{x}_{i},\mathbf{y}_{j})\\ &+ \frac{1}{m(m-1)}\displaystyle\sum_{i=1}^{m}\displaystyle\sum_{j\neq i}^{m}k(\mathbf{y}_{i},\mathbf{y}_{j}). \end{split} MMDu2​(P,Q,H)​=n(n−1)1​i=1∑n​j=i∑n​k(xi​,xj​)−nm2​i=1∑n​j=1∑m​k(xi​,yj​)+m(m−1)1​i=1∑m​j=i∑m​k(yi​,yj​).​

This distance between mean embeddings of features represents the distance between distributions in the input space. In other words, the smaller the MMD, the smaller the distance is between distributions in the input space. This can be used to find a model to fit the training data of the toy problem.

First, a Gaussian kernel is defined. A scale factor of 0.01 is chosen, as this value gave the best results later on. 

k = SqExponentialKernel() ∘ ScaleTransform(0.01)

Second, 100 circles are generated with 100 equally spaced points. Each circle has a different radius ranging from 15 to 25. To compute the MMD between these circles and the input data, several Gram matrices need to be computed. In the code below, K1 and K2 are the Gram matrices generated by the function "kernelmatrix" over the input data and a generated model, respectively. The coordinates of each data point need to be considered as one input value xxx in the Gaussian kernel, hence the use of "RowVecs". In K3 each element is a Gaussian kernel of two inputs, one being the coordinates of the input data and the other one being the coordinates of the generated model given a radius. With the computed Gram matrices, for each generated circle, an MMD can be calculated.

# Generate an array of 100 Radii ranging from 15 to 25
R2 = LinRange(15, 25, n)
# Assign an array for the MMDs
mmds = Array{Float64}(undef, 0, 1)
# Generate the kernelmatrix over the coordinates of the input data
K1 = kernelmatrix(k, RowVecs(X))
for i in 1:n
	# Generate a model circle consisting of 100 points with Radius i
	A = circle(100, R2[i])
	# Generate the kernelmatrix over the coordinates of the model circle
	K2 = kernelmatrix(k, RowVecs(A))
	# Generate the kernelmatrix over the coordinates of the input data 
	# and the model circle
	K3 = kernelmatrix(k, RowVecs(X), RowVecs(A))
	# Calculate the MMD
	mmd = mean(K1) - 2 * mean(K3) + mean(K2)
	# Add the MMD to the mmds array
	mmds = [mmds; mmd]
end

In this section, the kernel mean embedding of marginal distributions is extended to conditional distributions P(Y∣X)\mathbb P(Y|X)P(Y∣X) and P(Y∣X=x)\mathbb P(Y|X=\mathbf{x})P(Y∣X=x) for some x∈X\mathbf x \in \mathcal Xx∈X, capturing even more complex data [30] [33]. The figure below illustrates conditional mean embedding [1].

Consider the two positive definite kernels, k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R and l:Y×Y→Rl : \mathcal Y \times \mathcal Y \to \mathbb Rl:Y×Y→R for the respective domains of XXX and YYY, and the respective RKHSs H\mathcal HH and G\mathcal GG. To fully represent P(Y∣X)\mathbb P(Y|X)P(Y∣X), conditioning and conditional expectation need to be performed. Then, the conditional mean embeddings of the conditional distributions P(Y∣X)\mathbb P(Y|X)P(Y∣X) and P(Y∣X=x)\mathbb P(Y|X=\mathbf{x})P(Y∣X=x) can be defined as UY∣X:H→G\mathcal U_{Y|X}: \mathcal H \to \mathcal GUY∣X​:H→G and UY∣X∈G\mathcal U_{Y|X} \in \mathcal GUY∣X​∈G, such that:

UY∣x=EY∣X[φ(Y)∣X=x]=UY∣Xk(x,⋅),EY∣x[g(Y)∣X=x]=⟨g,UY∣x⟩G,∀g∈G. \begin{split} \mathcal U_{Y|\mathbf{x}} &= \mathbb E_{Y|X}[\varphi(Y)|X=\mathbf{x}] = \mathcal U_{Y|X}k(\mathbf{x},\cdot),\\ \mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] &= \langle g,\mathcal U_{Y|\mathbf{x}}\rangle_{\mathcal G}, \forall g \in \mathcal G. \end{split} UY∣x​EY∣x​[g(Y)∣X=x]​=EY∣X​[φ(Y)∣X=x]=UY∣X​k(x,⋅),=⟨g,UY∣x​⟩G​,∀g∈G.​

It is important to note that UY∣X\mathcal U_{Y|X}UY∣X​ is an operator from RKHS H\mathcal HH to G\mathcal GG, while UY∣x\mathcal U_{Y|\mathbf{x}}UY∣x​ is an element in RKHS G\mathcal GG. In the above equation it is stated that the conditional mean embedding of P(Y∣X=x)\mathbb P(Y|X=\mathbf{x})P(Y∣X=x) is the conditional expectation of the feature map of YYY given that X=xX = \mathbf{x}X=x. The operator UY∣X\mathcal U_{Y|X}UY∣X​ is the conditioning operation that when applied to ϕ(x)∈H\phi(x) \in \mathcal Hϕ(x)∈H delivers the conditional mean embedding UY∣x\mathcal U_{Y|x}UY∣x​. Equation 24 shows the reproducing property of UY∣x\mathcal U_{Y|x}UY∣x​: it should be a representer of conditional expectation in G\mathcal GG with regards to P(Y∣X=x)\mathbb P(Y|X=x)P(Y∣X=x).

Song et al. [30] [33] provide following definition: let CXX:H→H\mathcal C_{XX}: \mathcal H \to \mathcal HCXX​:H→H and CXY:H→G\mathcal C_{XY}: \mathcal H \to \mathcal GCXY​:H→G be the covariance operator on XXX and cross-covariance operator from XXX to YYY, respectively. Then, the conditional mean embeddings UY∣X\mathcal U_{Y|X}UY∣X​ and UY∣x\mathcal U_{Y|x}UY∣x​ are defined as:

UY∣X:=CXYCXX−1UY∣x:=CXYCXX−1k(x,⋅). \begin{split} \mathcal U_{Y|X} &:= \mathcal C_{XY}\mathcal C_{XX}^{-1}\\ \mathcal U_{Y|\mathbf{x}} &:= \mathcal{C}_{XY} \mathcal{C}_{XX}^{-1}k(\mathbf{x},\cdot). \end{split} UY∣X​UY∣x​​:=CXY​CXX−1​:=CXY​CXX−1​k(x,⋅).​

Fukumizu et al. [27] [34] state that if EY∣X[g(Y)∣X=⋅]∈H\mathbb E_{Y|X}[g(Y)|X=\cdot] \in \mathcal HEY∣X​[g(Y)∣X=⋅]∈H for any g∈Gg \in \mathcal Gg∈G, then

CXXEYX[g(Y)∣X=⋅]=CXYg. C_{XX}\mathbb E_{YX}[g(Y)|X=\cdot]=\mathcal{C}_{XY}g. CXX​EYX​[g(Y)∣X=⋅]=CXY​g.

Then it follows from the reproducing property of G\mathcal{G}G that:

EY∣x[g(Y)∣X=x]=⟨EY∣X[g(Y)∣X],k(x,⋅)⟩H. \mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] = \langle \mathbb E_{Y|X}[g(Y)|X],k(\mathbf{x},\cdot)\rangle_{\mathcal{H}}. EY∣x​[g(Y)∣X=x]=⟨EY∣X​[g(Y)∣X],k(x,⋅)⟩H​.

When equation 26 and equation 27 are combined and the conjugate transpose of CXX−1CXY\mathcal{C}_{XX}^{-1}\mathcal{C}_{XY}CXX−1​CXY​ is taken, the following is obtained:

EY∣x[g(Y)∣X=x]=⟨g,CYXCXX−1k(x,⋅)⟩G=⟨g,UY∣x⟩G. \mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] = \langle g,\mathcal{C}_{YX}\mathcal{C}_{XX}^{-1}k(\mathbf{x},\cdot)\rangle_{\mathcal{G}} = \langle g,\mathcal U_{Y|\mathbf{x}}\rangle_{\mathcal{G}}. EY∣x​[g(Y)∣X=x]=⟨g,CYX​CXX−1​k(x,⋅)⟩G​=⟨g,UY∣x​⟩G​.

In an infinite RKHS, CXX−1C_{XX}^{-1}CXX−1​ does not exist. Hence, a regularized version of equation 25 is used:

UY∣X:=CXY(CXX+λI)−1, \mathcal U_{Y|X} := \mathcal C_{XY}(\mathcal C_{XX} + \lambda \mathcal I)^{-1}, UY∣X​:=CXY​(CXX​+λI)−1,

where λ>0\lambda > 0λ>0 is the regularization parameter and I\mathcal II is the identity operator in H\mathcal HH. Fukumizu et al. So, the conditional mean embedding of μY∣x\mu_{Y|\mathbf{x}}μY∣x​ is estimated by [33] :

Learning on distributional data

The conditional mean embedding has a natural interpretation as a solution to a vector-valued regression problem, observed first by Zhang et al. [35] and later by Grünewälder et al. [36]. As discussed earlier, the conditional mean embedding is defined via EY∣x[g(Y)∣X=x]=⟨g,μ^Y∣x⟩G\mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] = \langle g, \hat{\mu}_{Y|\mathbf{x}}\rangle_{\mathcal{G}}EY∣x​[g(Y)∣X=x]=⟨g,μ^​Y∣x​⟩G​, for every x∈X\mathbf x \in \mathcal Xx∈X, μ^Y∣x\hat{\mu}_{Y|\mathbf{x}}μ^​Y∣x​ is a function on Y\mathcal YY and consequently defines a mapping from X\mathcal XX to G\mathcal GG. Moreover, the empirical estimator μ^Y∣x=Φ(K+nλIn)−1kx\hat{\mu}_{Y|\mathbf{x}} = \Phi(\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}}μ^​Y∣x​=Φ(K+nλIn​)−1kx​, suggests that the conditional mean embedding is the solution to an underlying regression problem.

Consider the i.i.d. sample (x1,z1),...,(xn,zn)∈X×G(\mathbf{x}_{1},\mathbf{z}_{1}),...,(\mathbf{x}_{n},\mathbf{z}_{n}) \in \mathcal X \times \mathcal G(x1​,z1​),...,(xn​,zn​)∈X×G, a vector-valued regression problem can be formulated as:

ε^λ(f)=∑i=1n∥zi−f(xi)∥G2+λ∥f∥HΓ2, \hat{\varepsilon}_{\lambda}(f) = \displaystyle\sum_{i=1}^{n} {\|\mathbf{z}_{i} - f(\mathbf{x}_{i})\|}_{\mathcal G}^{2} + \lambda {\|f\|}_{\mathcal {H}_{\Gamma}}^{2}, ε^λ​(f)=i=1∑n​∥zi​−f(xi​)∥G2​+λ∥f∥HΓ​2​,

where G\mathcal GG is a Hilbert space, HΓ\mathcal {H}_{\Gamma}HΓ​ an RKHS of vector-valued functions from X\mathcal XX to G\mathcal GG, and ε^λ\hat{\varepsilon}_{\lambda}ε^λ​ is error of the associated regression problem [37]. Grünewälder et al. [36] states that by minimizing the optimization in equation 32, μ^Y∣x\hat{\mu}_{Y|\mathbf{x}}μ^​Y∣x​ can be obtained. So, the natural optimization problem for the conditional mean embedding is to find a function μ:X→G\mu : \mathcal X \to \mathcal Gμ:X→G that minimizes an objective. This objective can be bounded from above by a surrogate loss function, which can be described by its empirical counterpart [36] :

ε^S[μ]=∑i=1n∥l(yi,⋅)−μ(xi)∥G2+λ∥μ∥HΓ2. \hat{\varepsilon}_{\mathcal S}[\mu] = \displaystyle\sum_{i=1}^{n} {\|l(\mathbf{y}_i,\cdot) - \mu(\mathbf{x}_i)\|}_{\mathcal G}^{2} + \lambda {\|\mu\|}_{\mathcal {H}_{\Gamma}}^{2}. ε^S​[μ]=i=1∑n​∥l(yi​,⋅)−μ(xi​)∥G2​+λ∥μ∥HΓ​2​.

The added regularization term provides a well-posed problem and prevents overfitting. Interpreting the conditional mean embedding as a solution to a vector-valued regression problem gives the advantage of being able to use cross-validation or model selection, due to the well-defined loss function. Since G\mathcal GG is assumed to be finite-dimensional, the conditional mean embedding is the ridge regression of the feature vectors. Consider βλ^:=(K+nλIn)−1kx\hat{\mathbf{ \beta}_{\lambda}} := (\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}}βλ​^​:=(K+nλIn​)−1kx​, in a ridge regression context the hat matrix Hλ\mathbf{H}_{\lambda}Hλ​ is:

Hλkx=k^x=Φβ^λHλ=K(K+λI)−1. \begin{split} \mathbf{H}_{\lambda}\mathbf{k}_{\mathbf{x}} &= \hat{\mathbf{k}}_{\mathbf{x}}\\ &= \Phi \hat{\mathbf{\beta}}_{\lambda}\\ \mathbf{H}_{\lambda} &= \mathbf{K}(\mathbf{K} + \lambda \mathbf{I})^{-1}. \end{split} Hλ​kx​Hλ​​=k^x​=Φβ^​λ​=K(K+λI)−1.​

Using leave-one-out cross-validation (LOOCV), the estimated conditional embedding is defined as [38] :

μ^Y∣xLOOCV=(I−diag(Hλ))−1(Hλ−diag(Hλ))Φ, \hat{\mathbf{\mu}}_{Y|\mathbf{x}}^{LOOCV} = (\mathbf{I} - diag(\mathbf{H}_{\lambda}))^{-1}(\mathbf{H}_{\lambda} - diag(\mathbf{H}_{\lambda}))\Phi, μ^​Y∣xLOOCV​=(I−diag(Hλ​))−1(Hλ​−diag(Hλ​))Φ,

with diag(⋅)diag(\cdot)diag(⋅) the diagonal matrix. It's important to note that using the above equation, all LOOCV conditional embeddings can be calculated at once using matrix multiplications.

To interpret the obtained results, the underlying distributions needs to be recovered from the embeddings, which is the topic of the next section.

Recovering distributions from RKHS embeddings

Recovering information of P\mathbb PP from the kernel mean embedding μP\mu_{\mathbb P}μP​ is known as the distributional pre-image problem [39] [40]. In this context, objects in the input space which correspond with a specific kernel mean embedding in a feature space, are looked for. So, the goal is to recover useful information of an unknown underlying distribution from its estimated embedding. Consider Pθ\mathbb{P}_{\theta}Pθ​ an arbitrary distribution parameterized by θ\mathbf{\theta}θ and μPθ\mu_{\mathbb{P}_{\theta}}μPθ​​ its mean embedding in H\mathcal HH. By solving the following minimization problem, Pθ\mathbb{P}_{\theta}Pθ​ can be found:

θ^=arg min⁡θ∈Θ∥μ^Y−μPθ∥H2=arg min⁡θ∈Θ⟨μ^Y,μ^Y⟩−2⟨μ^Y,μPθ⟩+⟨μPθ,μPθ⟩, \begin{split} \hat{\mathbf{\theta}} &= \argmin_{\theta \in \Theta}{\|\hat{\mu}_{Y} - \mu_{\mathbb{P}_{\theta}}\|}_{\mathcal H}^{2}\\ &= \argmin_{\theta \in \Theta} \langle \hat{\mu}_{Y},\hat{\mu}_{Y}\rangle -2\langle \hat{\mu}_{Y}, \mu_{\mathbb{P}_{\theta}}\rangle + \langle \mu_{\mathbb{P}_{\theta}}, \mu_{\mathbb{P}_{\theta}}\rangle, \end{split} θ^​=θ∈Θargmin​∥μ^​Y​−μPθ​​∥H2​=θ∈Θargmin​⟨μ^​Y​,μ^​Y​⟩−2⟨μ^​Y​,μPθ​​⟩+⟨μPθ​​,μPθ​​⟩,​

where θ\thetaθ is subject to appropriate constraints. As seen earlier, equation 36 describes the maximum mean discrepancy (MMD). As a reminder, this is the distance between mean embedding of features, which represents the distances between distributions in the input space. ⟨μ^Y,μ^Y⟩\langle \hat{\mu}_{Y},\hat{\mu}_{Y}\rangle⟨μ^​Y​,μ^​Y​⟩ is constant and is left out of the minimization. Assume μPθ=∑i=1nαiφ(yi)\mu_{\mathbb{P}_{\theta}} = \sum_{i=1}^{n}{\alpha}_i\varphi(\mathbf{y}_i)μPθ​​=∑i=1n​αi​φ(yi​) for some α∈Δn−1\alpha \in \Delta^{n-1}α∈Δn−1, i.e., Pθ\mathbb{P}_{\theta}Pθ​ is a histogram. Let Lij=l(yi,yj)\mathbf{L}_{ij}=l(\mathbf{y}_{i}, \mathbf{y}_{j})Lij​=l(yi​,yj​), then ⟨μPθ,μPθ⟩=α′(L+λI)α\langle \mu_{\mathbb{P}_{\theta}}, \mu_{\mathbb{P}_{\theta}}\rangle = \alpha'(\mathbf{L} + \lambda\mathbf{I})\alpha⟨μPθ​​,μPθ​​⟩=α′(L+λI)α. Because of the addition of the regularization term λ\lambdaλ, the optimization can be interpreted as a standard quadratic programming problem. The last component of equation 36 is ⟨μ^Y,μPθ⟩\langle \hat{\mu}_{Y}, \mu_{\mathbb{P}_{\theta}}\rangle⟨μ^​Y​,μPθ​​⟩, which corresponds to the dot product of Pθ\mathbb{P}_{\theta}Pθ​ and the LOOCV estimated conditional embedding of equation 35. This results being able to write equation 36 as:

α^=arg min⁡α∈Δn−1α′(L+λI)α−2α⋅μ^Y∣xLOOCV. \hat{\mathbf{\alpha}} = \argmin_{\alpha \in \Delta^{n-1}}\mathbf{\alpha}'(\mathbf{L} + \lambda\mathbf{I})\mathbf{\alpha} - 2\mathbf{\alpha}\cdot \hat{\mathbf{\mu}}_{Y|\mathbf{x}}^{LOOCV}. α^=α∈Δn−1argmin​α′(L+λI)α−2α⋅μ^​Y∣xLOOCV​.

When using a Gaussian kernel on Euclidean space, Kanagawa and Fukumizu (2014) [40] show that under some mild conditions certain information can be recovered from P\mathbb{P}P. This information mainly consists of the moments of the probability distribution and measures on intervals. The most important thing is that the density of P\mathbb{P}P can be estimated from the kernel mean embedding, without any parametric assumption on P\mathbb{P}P.
