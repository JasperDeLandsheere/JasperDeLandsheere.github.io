Kernel Methods

Inner products serve as a powerful tool in many established machine learning algorithms, such as principal component analysis (PCA)[2] [3], perceptron [4] [5], and support vector machine (SVM) [6]. These algorithms consider the data, e.g., x,x′∈X\mathbf{x},\mathbf{x}' \in \mathcal Xx,x′∈X, with X\mathcal XX a non empty set, through their inner product ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}' \rangle⟨x,x′⟩, which can be interpreted as a similarity measure between x\mathbf{x}x and x′\mathbf{x}'x′. But real-life data is often complex and the class of linear functions induced by the inner products might prove to be insufficient. The aim of kernel methods is to handle complex data which can't be linearly evaluated, by replacing ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}' \rangle⟨x,x′⟩ with some other (non-linear) similarity measure.

Naturally, an extension of ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}' \rangle⟨x,x′⟩ can be made by explicitly applying a non-linear transformation:

ϕ:X→F,x↦ϕ(x), \phi:\mathcal X\to \mathcal F,\\ x \mapsto \phi (x), ϕ:X→F,x↦ϕ(x),

from X\mathcal XX to the high-dimensional feature space F\mathcal FF. In this new feature space the inner product can be evaluated:

k(x,x′):=⟨ϕ(x),ϕ(x′)⟩F, k(\mathbf{x},\mathbf{x}') := \langle \phi(\mathbf{x}), \phi(\mathbf{x}') \rangle_{\mathcal{F}}, k(x,x′):=⟨ϕ(x),ϕ(x′)⟩F​,

where ⟨⋅,⋅⟩F\langle \cdot, \cdot \rangle_{\mathcal{F}}⟨⋅,⋅⟩F​ is the inner product of F\mathcal FF and ϕ\phiϕ is called the feature map. kkk is the kernel function which defines a non-linear similarity measure between x\mathbf{x}x and x′\mathbf{x}'x′. By substituting ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}' \rangle⟨x,x′⟩ with ⟨ϕ(x),ϕ(x′)⟩F\langle \phi(\mathbf{x}), \phi(\mathbf{x}') \rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​ a non-linear extension of the algorithms that consider data through ⟨x,x′⟩\langle \mathbf{x}, \mathbf{x}' \rangle⟨x,x′⟩, can be made. So, a linear algorithm in F\mathcal FF corresponds to a non-linear counterpart in the original input space. It is important to note that the algorithm stays the same, only the space in which the algorithm operates, changes.

The core idea of kernel methods is taking data which lives in an input space where it's not easy to perform machine learning and transform this data to a higher dimensional space where effective use of learning algorithms can be made.

As seen in the above example, to evaluate equation 2 one needs to work in two steps: i) explicitly constructing the feature maps ϕ(x)\phi(\mathbf{x})ϕ(x), and ii) evaluating the inner product ⟨ϕ(x),ϕ(x′)⟩F\langle \phi(\mathbf{x}), \phi(\mathbf{x}') \rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​. This can become a problem when ϕ(x)\phi(\mathbf{x})ϕ(x) defines a computationally expensive transformation to a high-dimensional featue space. Fortunately, there exists a solution to this by evaluating ⟨ϕ(x),ϕ(x′)⟩F\langle \phi(\mathbf{x}), \phi(\mathbf{x}') \rangle_{\mathcal{F}}⟨ϕ(x),ϕ(x′)⟩F​ without explicitly constructing the feature maps. This is a core idea of kernel methods and is often called "the kernel trick" in the machine learning community [1]. A visual representation of the kernel trick is illustrated below. For the above mentioned example one can just consider k(x,x′)=⟨x,x′⟩2k(\mathbf{x},\mathbf{x}') = {\langle \mathbf{x}, \mathbf{x}' \rangle}^2k(x,x′)=⟨x,x′⟩2 rather than calculating the feature maps explicitly.

What are the requirements for the kernel trick to do its magic? If kkk is positive definite there always exists a feature map ϕ:X→F\phi : \mathcal X \to \mathcal Fϕ:X→F such that k(x,x′)=⟨ϕ(x),ϕ(x′)⟩Fk(\mathbf{x},\mathbf{x}') = \langle \phi(\mathbf{x}), \phi(\mathbf{x}') \rangle_{\mathcal{F}}k(x,x′)=⟨ϕ(x),ϕ(x′)⟩F​ [7] without the explicit construction of ϕ\phiϕ [6] [8]. So, the kernel function is used as a way to calculate an inner product ⟨ϕ(x),ϕ(x′)⟩\langle \phi(\mathbf{x}), \phi(\mathbf{x}') \rangle⟨ϕ(x),ϕ(x′)⟩ in a high-dimensional feature space H\mathcal HH for some data points x,x′∈X\mathbf{x},\mathbf{x}' \in \mathcal Xx,x′∈X. The collection of all these pairwise inner products within the set of data vectors x\mathbf{x}x is defined as the n×nn \times nn×n-Gram or kernel matrix Kij:=k(xi,xj)K_{ij} := k(\mathbf{x}_{i},\mathbf{x}_{j})Kij​:=k(xi​,xj​). The comparison function kkk is a positive definite kernel on X\mathcal XX if it is symmetric, i.e., k(x,x′)=k(x′,x)k(\mathbf{x}, \mathbf{x}') = k(\mathbf{x}',\mathbf{x})k(x,x′)=k(x′,x), and the Gram matrix is positive definite:

∑i=1n∑n=1jcicjk(xi,xj)≥0, \displaystyle\sum_{i=1}^{n}\displaystyle\sum_{n=1}^{j} c_{i}c_{j}k(\mathbf{x}_{i},\mathbf{x}_{j}) \geq 0, i=1∑n​n=1∑j​ci​cj​k(xi​,xj​)≥0,

for any n∈Nn \in \mathbb Nn∈N, all finite sequences of points (x1,...,xn)∈Xn(x_{1},...,x_{n}) \in \mathcal X^{n}(x1​,...,xn​)∈Xn and any nnn real-valued coefficients (c1,...,cn)∈Rn(c_{1},...,c_{n}) \in \mathbb R^{n}(c1​,...,cn​)∈Rn [9] [10].

Another important property of a positive definite kernel is that it induces a space of functions from X\mathcal XX to R\mathbb RR called a reproducing kernel Hilbert space (RKHS) H\mathcal HH, which is why the kernel is also called a reproducing kernel [7]. It's important to note that the RKHS is a space of functions from X\mathcal XX to R\mathbb RR. In other words, each data point x\mathbf{x}x in X\mathcal XX will be represented by a function ϕ(x)\phi(\mathbf{x})ϕ(x) in H\mathcal HH.

An RKHS has two properties important to kernel mean embedding: (i) for any x∈X\mathbf{x} \in \mathcal Xx∈X, the function k(x,⋅):y↦k(x,y)k(\mathbf{x},\cdot):\mathbf{y} \mapsto k(\mathbf{x},\mathbf{y})k(x,⋅):y↦k(x,y) is an element of H\mathcal HH. So, whenever the kernel kkk is used, the feature space F\mathcal FF is the RKHS H\mathcal HH associated with this kernel. This can be considered as the canonical feature map:

k:X→H⊂RX,x↦k(x,⋅), k:\mathcal X \to \mathcal H \subset {\mathbb R}^{\mathcal X},\\ x \mapsto k(\mathbf{x},\cdot), k:X→H⊂RX,x↦k(x,⋅),

where RX\mathbb R^{\mathcal X}RX denotes the vector space of functions from X\mathcal XX to R\mathbb RR; (ii) a function k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R is called a reproducing kernel of H\mathcal HH if k(⋅,x)∈Hk(\cdot,\mathbf{x}) \in \mathcal Hk(⋅,x)∈H for all x∈X\mathbf{x} \in \mathcal Xx∈X and the reproducing property

f(x)=⟨f,k(⋅,x)⟩H f(\mathbf{x}) = \langle f,k(\cdot,\mathbf{x}) \rangle_{\mathcal H} f(x)=⟨f,k(⋅,x)⟩H​

holds for all f∈Hf \in \mathcal Hf∈H and all x∈Xx \in \mathcal Xx∈X. In particular: if f(x′=k(x′,⋅))f(\mathbf{x'} = k(\mathbf{x}',\cdot))f(x′=k(x′,⋅)) for some x′∈X\mathbf{x}' \in \mathcal Xx′∈X, k(x,x′)=⟨k(x,⋅),k(x′,⋅)⟩Hk(\mathbf{x},\mathbf{x}')=\langle k(\mathbf{x},\cdot),k(\mathbf{x}',\cdot) \rangle_{\mathcal H}k(x,x′)=⟨k(x,⋅),k(x′,⋅)⟩H​.

Aronszajn (1950): “There is a one-to-one correspondence between the reproducing kernel kkk and the RKHS H\mathcal HH”.

The kernel trick not only delivers powerful (non-linear) learning algorithms, but also paves the path for domain experts to invent certain kernels which are suitable for specific applications. The kernel trick does not only apply to Euclidean data, but also to non-Euclidean structured data, functional data, and other domains on which a positive definite kernel may be applied [8] [11]. Various kernels have been proposed in various application domains [12] and for different types of data, such as strings, graphs and trees [8] [11] [13].

The kernel used in this thesis is called the Gaussian kernel, which is a member of the class of kernels called radial basis functions (RBFs):

kRBF(x,x′)=exp(−∥x−x′∥22σ2), k^{RBF}(\mathbf{x},\mathbf{x}') = exp(-\frac{\|\mathbf{x}-\mathbf{x}'\|^{2}}{2{\sigma}^{2}}), kRBF(x,x′)=exp(−2σ2∥x−x′∥2​),

with σ>0\sigma > 0σ>0 the bandwidth parameter. The Gram matrix of the Gaussian kernel becomes a matrix of ones for σ→∞\sigma \to \inftyσ→∞ and an identity matrix for σ→0\sigma \to 0σ→0. Which means for the former that all instances are the same, and for the latter that all instances are completely unique, making it a relevant interpretation as a similarity measure [14]. RBF kernels are stationary kernels, which means that they can be described as functions of the differences of their input. RBF kernels are also universal kernels, which means they can represent any smooth function with a high degree of accuracy, assuming being able to find the right bandwidth parameter [12] [15].

For additional information on the properties of (reproducing kernel) Hilbert spaces and the important theorems of Mercer and Bochner, the reader is advised to read Muandet et al. [1], Mercer [10], and Bochner [16], respectively. For examples of learning algorithms that use the implicit representation of data points in kernel methods, such as support vector machine (SVM), gaussian process (GP), and neural tangent kernel (NTK), the reader is referred to read Steinwart & Christmann [17], Rasmussen [18], and Jacot et al. [19], respectively.

Kernel Mean Embedding of Marginal Distributions

Having reviewed the above section on kernel methods one could wonder the practicality of extending kernel methods from individual data points to probability distributions. In many real-life learning problems, however, it could be argued that it is more appropriate to represent the training data as probability distributions rather than individual data points. For example, in many situations data is missing or uncertain. As a specific example, gene expression data originating from microarray experiments are known to be very noisy, due to different sources of variabilities [20]. To battle this, each array can be represented as a probability distribution. Another reason for the preference of probability distributions can be computational challenges when dealing with large amounts of training data [21].

Let k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R be a real-valued positive definite kernel associated with the Hilbert space H\mathcal HH, with X\mathcal XX a non-empty set. The reproducing property lets the kernel evaluation be interpreted as an inner product in H\mathcal HH induced by a map from X\mathcal XX into H\mathcal HH

x↦k(x,⋅). \mathbf{x} \mapsto k(\mathbf{x,\cdot}). x↦k(x,⋅).

Basically, k(x,⋅)k(\mathbf{x,\cdot})k(x,⋅) is a high-dimensional representer of x\mathbf{x}x and because of the reproducing property k(x,⋅)k(\mathbf{x,\cdot})k(x,⋅) is also a representer of evaluation of any function in H\mathcal HH on the data point x\mathbf{x}x. This lets feature map ϕ\phiϕ to be extended to the space of probability distributions through the mapping of µµµ which defines the representer in H\mathcal HH of any distribution P\mathbb PP [21] :

µ:MX1(X)→HP↦∫Xk(x,⋅)dP(x), µ: M_X^{1}(\mathcal X) \to \mathcal H\\ \mathbb P \mapsto \int_{\mathcal X} k(\mathbf{x},\cdot) \mathrm{d\mathbb P}(\mathbf{x}), µ:MX1​(X)→HP↦∫X​k(x,⋅)dP(x),

with MX1(X)M_X^{1}(\mathcal X)MX1​(X) the space of probability measures over a measurable space X\mathcal XX. The distribution P\mathbb PP is transformed into an element, the mean embedding µP{µ}_{\mathbb P}µP​, in an RKHS corresponding to the positive definite kernel kkk, hence the name kernel mean embedding:

ϕ(P)=µP:=EX∼P[k(X,⋅)]=∫Xk(x,⋅)dP(x). \phi(\mathbb P) = µ_{\mathbb P} := \mathbb E_{X\sim \mathbb P}[k(X,\cdot)] = \int_{\mathcal X} k(\mathbf{x},\cdot) \mathrm{d\mathbb P}(\mathbf{x}). ϕ(P)=µP​:=EX∼P​[k(X,⋅)]=∫X​k(x,⋅)dP(x).

The element is the expected value in the RKHS and since P\mathbb PP is a probability density distribution, it can be interpreted as an integral [22] [23]. A visual representation of this mean embedding:

One can wonder how much information about the distribution P\mathbb PP this kernel mean embedding can capture. This depends on the used kernel and can range from only the first moment of P\mathbb PP to all information of P\mathbb PP. Consider following examples with their corresponding captured information:

  • k(x,x′)=⟨x,x′⟩k(\mathbf{x},\mathbf{x}')=\langle \mathbf{x},\mathbf{x}' \ranglek(x,x′)=⟨x,x′⟩ : the first moment of P\mathbb PP

  • k(x,x′)=(⟨x,x′⟩+1)pk(\mathbf{x},\mathbf{x}')={(\langle \mathbf{x},\mathbf{x}' \rangle+1)}^{p}k(x,x′)=(⟨x,x′⟩+1)p: moments of P\mathbb PP up to order p∈Np \in \mathbb Np∈N

  • k(x,x′)k(\mathbf{x},\mathbf{x}')k(x,x′) is universal/characteristic: all information of P\mathbb PP

The first and second example are called the linear and polynomial kernel, respectively. The linear kernel, which equals the computation of the inner product, lets µPµ_{\mathbb P}µP​ retain the first moment of P\mathbb PP. For the polynomial kernel of order ppp, the mean map equals up to the ppp-th moment of P\mathbb PP. For some other explicit examples the reader is referred to Smola et al. [22], Fukumizu et al. [24], Sriperumbudur et al. [25], Gretton et al. [26], and Schölkopf et al. [27].

In the third example, characteristic and universal kernels are mentioned. Chracteristic kernels are a class of kernel functions for which the kernel mean embedding captures all information about the distribution P\mathbb PP. A kernel kkk is characteristic if the map P↦µP\mathbb P \mapsto µ_{\mathbb P}P↦µP​ is injective, i.e., ∥µP−µQ∥H=0{\|µ_{\mathbb P} - µ_{\mathbb Q} \|}_{\mathcal H} = 0∥µP​−µQ​∥H​=0 if and only if P=Q\mathbb P = \mathbb QP=Q [28]. The Gaussian kernel used in this thesis is a characteristic kernel [24]. It must be noted that all universal kernels are characteristic, but characteristic kernels may not be universal [26] [29].

This injectivity of the map P↦µP\mathbb P \mapsto µ_{\mathbb P}P↦µP​ ensures that the RKHS embedding is suitibale for regression problems, since each element in the feature space corresponds to one unique distribution in the original space. In other words, no information is lost when mapping the distribution into the RKHS.

It is important to note that in practice, the true distribution of P\mathbb PP is often unkown, and only an i.i.d. sample x1,...,xnx_{1},...,x_{n}x1​,...,xn​ from P\mathbb PP is avaible. µPµ_{\mathbb P}µP​ can be estimated by taking an empirical average:

µ^P:=1n∑i=1nk(xi,⋅), \hat{µ}_{\mathbb P} := \frac{1}{n}\displaystyle\sum_{i=1}^{n}k(\mathbf{x}_{i}, \cdot), µ^P​:=n1​i=1∑n​k(xi​,⋅),

with µ^P\hat{µ}_{\mathbb P}µ^P​ an unbiased estimate of µP{µ}_{\mathbb P}µP​. The weak law of large numbers indicates that µ^P\hat{µ}_{\mathbb P}µ^P​ converges to the true mean embedding µP{µ}_{\mathbb P}µP​ [30]. In this thesis, the data is interpreted as a probability mass distribution of X\mathbf XX. For example, P^:=1n∑i=1nδxi\hat{\mathbb P} := \frac{1}{n}\sum_{i=1}^{n}{\delta}_{\mathbf{x}_i}P^:=n1​∑i=1n​δxi​​, with δx{\delta}_{\mathbf{x}}δx​ the Dirac measure defined for x∈X\mathbf x \in \mathcal Xx∈X, such that:

µ^P:=∑i=1nwik(xi,⋅), \hat{µ}_{\mathbb P} := \displaystyle\sum_{i=1}^{n}w_{i}k(\mathbf{x}_{i}, \cdot), µ^P​:=i=1∑n​wi​k(xi​,⋅),

with w=[wi]∈Δn−1\mathbf w = [w_i] \in \Delta^{n-1}w=[wi​]∈Δn−1, i.e., a histogram with weights subject to the constraint ∑inwi=1\sum_{i}^{n}w_i = 1∑in​wi​=1 and wi>0w_i > 0wi​>0 [31].

In the next section kernel mean embedding is used to define a metric for probability functions, called the maximum mean discrepancy (MMD). This metric is very important for solving problems in statistics and machine learning when handling distributions.

Toy problem 1: Inference (using Maximum mean discrepancy)

Consider following arbitrary toy problem: given noisy data points which lay in a circular shape, is it possible to find a model fit of 100 equally spaced points which lay on a circle with a certain radius that represents the input data?

This problem can be solved using kernel mean embedding, particularly using the maximum mean discrepancy (MMD). The maximum mean discrepancy corresponds to the RKHS distance between mean embeddings [26] [32] :

MMD2(P,Q,H)=∥µP−µQ∥H2=∥µP∥H−2⟨µP,µQ⟩H+∥µQ∥H. MMD^{2}(\mathbb P, \mathbb Q, \mathcal H) = {\|µ_{\mathbb P} - µ_{\mathbb Q}\|}_{\mathcal H}^{2} = {\|µ_{\mathbb P}\|}_{\mathcal H} -2{\langle µ_{\mathbb P}, µ_{\mathbb Q} \rangle}_{\mathcal H} + {\|µ_{\mathbb Q}\|}_{\mathcal H}. MMD2(P,Q,H)=∥µP​−µQ​∥H2​=∥µP​∥H​−2⟨µP​,µQ​⟩H​+∥µQ​∥H​.

Given {xi}i=1n∼P\{x_i\}_{i=1}^n \sim \mathbb P{xi​}i=1n​∼P and {yj}j=1n∼Q\{y_j\}_{j=1}^n \sim \mathbb Q{yj​}j=1n​∼Q, the empirical MMD is [32] :

MMDu2(P,Q,H)=1n(n−1)∑i=1n∑j≠ink(xi,xj)−2nm∑i=1n∑j=1mk(xi,yj)+1m(m−1)∑i=1m∑j≠imk(yi,yj). MMD_u^{2}(\mathbb P, \mathbb Q, \mathcal H) = \frac{1}{n(n-1)}\displaystyle\sum_{i=1}^{n}\displaystyle\sum_{j\neq i}^{n}k(\mathbf{x}_{i},\mathbf{x}_{j}) - \frac{2}{nm}\displaystyle\sum_{i=1}^{n}\displaystyle\sum_{j = 1}^{m}k(\mathbf{x}_{i},\mathbf{y}_{j})+ \frac{1}{m(m-1)}\displaystyle\sum_{i=1}^{m}\displaystyle\sum_{j\neq i}^{m}k(\mathbf{y}_{i},\mathbf{y}_{j}). MMDu2​(P,Q,H)=n(n−1)1​i=1∑n​j=i∑n​k(xi​,xj​)−nm2​i=1∑n​j=1∑m​k(xi​,yj​)+m(m−1)1​i=1∑m​j=i∑m​k(yi​,yj​).

This distance between mean embeddings of features represents the distance between distributions in the input space. In other words, the smaller the MMD, the smaller the distance is between distributions in the input space. This can be used to find a model to fit the training data of the toy problem.

First, a Gaussian kernel is defined. In this example the KernelFunctions.lj package is used. The scaleTransfrom parameter is the inverse of the length scale (i.e. bandwidth parameter of the Gaussian kernel). 

k = SqExponentialKernel() ∘ ScaleTransform(0.01)

Second, 100 circles are generated with 100 equally spaced points. Each circle has a different radius ranging from 15 to 25. To compute the MMD between these circles and the input data, several Gram matrices need to be computed. In the code below, K1 and K2 are the Gram matrices generated by the function "kernelmatrix" over the input data and a generated model, respectively. The coordinates of each data point need to be considered as one input value xxx in the Gaussian kernel, hence the use of "RowVecs". In K3 each element is a Gaussian kernel of two inputs, one being the coordinates of the input data and the other one being the coordinates of the generated model given a radius. With the computed Gram matrices, for each generated circle, an MMD can be calculated.

R2 = LinRange(15, 25, n) # Generate an array of n radii ranging from 15 to 25
mmds = Array{Float64}(undef, 0, 1) # Assign an array for the maximum mean discrepancies
K1 = kernelmatrix(k, RowVecs(X)) # Generate the Gram matrix of the input data
for i in 1:n
	A = circle(100, R2[i]) # Generate a model for given radius
	K2 = kernelmatrix(k, RowVecs(A)) # Generate the Gram matrix of the model data
	K3 = kernelmatrix(k, RowVecs(X), RowVecs(A)) # Generate the Gram matrix of the input and model data 
	mmd = mean(K1) - 2 * mean(K3) + mean(K2) a # Calculate the MMD 
	mmds = [mmds; mmd] # Add the MMD to the array of MMDs
end

The obtained MMDs can be plotted versus the radius of each corresponding generated circle:

In the code below, a value for the radius of the best model fitted on the input data can be obtained. 

index = argmin(mmds) # Find the index of the minimum value in mmds
minimum(mmds) # The minimum value in mmds
R2[index] # The corresponding radius of the minimum MMD

In the graph below, the obtained model is plotted. The distance between the embedding of this model and the embedding of the input data is the smallest out of all generated circles, meaning the model fits the input data the best in the input space.

Kernel Mean Embedding of Conditional Distributions

In this section, the kernel mean embedding of marginal distributions is extended to conditional distributions P(Y∣X)\mathbb P(Y|X)P(Y∣X) and P(Y∣X=x)\mathbb P(Y|X=\mathbf{x})P(Y∣X=x) for some x∈X\mathbf x \in \mathcal Xx∈X, capturing even more complex data [31] [33]. The figure below illustrates conditional mean embedding [1].

Consider the two positive definite kernels, k:X×X→Rk : \mathcal X \times \mathcal X \to \mathbb Rk:X×X→R and l:Y×Y→Rl : \mathcal Y \times \mathcal Y \to \mathbb Rl:Y×Y→R for the respective domains of XXX and YYY, and the respective RKHSs H\mathcal HH and G\mathcal GG. To fully represent P(Y∣X)\mathbb P(Y|X)P(Y∣X), conditioning and conditional expectation need to be performed. Then, the conditional mean embeddings of the conditional distributions P(Y∣X)\mathbb P(Y|X)P(Y∣X) and P(Y∣X=x)\mathbb P(Y|X=\mathbf{x})P(Y∣X=x) can be defined as UY∣X:H→G\mathcal U_{Y|X}: \mathcal H \to \mathcal GUY∣X​:H→G and UY∣X∈G\mathcal U_{Y|X} \in \mathcal GUY∣X​∈G, such that:

UY∣x=EY∣X[φ(Y)∣X=x]=UY∣Xk(x,⋅),EY∣X[g(Y)∣X=x]=⟨g,UY∣x⟩G,∀g∈G. \mathcal U_{Y|\mathbf{x}} = \mathbb E_{Y|X}[\varphi(Y)|X=\mathbf{x}] = \mathcal U_{Y|X}k(\mathbf{x},\cdot),\\ \mathbb E_{Y|X}[g(Y)|X=\mathbf{x}] = \langle g,\mathcal U_{Y|\mathbf{x}} \rangle_{\mathcal G}, \forall g \in \mathcal G. UY∣x​=EY∣X​[φ(Y)∣X=x]=UY∣X​k(x,⋅),EY∣X​[g(Y)∣X=x]=⟨g,UY∣x​⟩G​,∀g∈G.

UY∣X\mathcal U_{Y|X}UY∣X​ is an operator from RKHS H\mathcal HH to G\mathcal GG, and UY∣x\mathcal U_{Y|\mathbf{x}}UY∣x​ is an element in RKHS G\mathcal GG. In the above equation it is stated that the conditional mean embedding of P(Y∣X=x)\mathbb P(Y|X=\mathbf{x})P(Y∣X=x) is the conditional expectation of the feature map of YYY given that X=xX = \mathbf{x}X=x. The operator UY∣X\mathcal U_{Y|X}UY∣X​ is the conditioning operation that when applied to ϕ(x)∈H\phi(x) \in \mathcal Hϕ(x)∈H delivers the conditional mean embedding UY∣x\mathcal U_{Y|x}UY∣x​. Equation 15 shows the reproducing property of UY∣x\mathcal U_{Y|x}UY∣x​: it should be a representer of conditional expectation in G\mathcal GG with regards to P(Y∣X=x)\mathbb P(Y|X=x)P(Y∣X=x).

Song et al. [31] [33] provide following definition: let CXX:H→H\mathcal C_{XX}: \mathcal H \to \mathcal HCXX​:H→H and CXY:H→G\mathcal C_{XY}: \mathcal H \to \mathcal GCXY​:H→G be the covariance operator on XXX and cross-covariance operator from XXX to YYY, respectively. Then, the conditional mean embedding UY∣X\mathcal U_{Y|X}UY∣X​ and UY∣x\mathcal U_{Y|x}UY∣x​ are defined as:

UY∣X:=CXYCXX−1UY∣x:=CXYCXX−1k(x,⋅). \mathcal U_{Y|X} := \mathcal C_{XY}\mathcal C_{XX}^{-1}\\ \mathcal U_{Y|\mathbf{x}} := \mathcal C_{XY}\mathcal C_{XX}^{-1}k(\mathbf{x},\cdot). UY∣X​:=CXY​CXX−1​UY∣x​:=CXY​CXX−1​k(x,⋅).

Fukumizu et al. [28] [34] state that if EY∣X[g(Y)∣X=⋅]∈H\mathbb E_{Y|X}[g(Y)|X=\cdot] \in \mathcal HEY∣X​[g(Y)∣X=⋅]∈H for any g∈Gg \in \mathcal Gg∈G, then

CXXEY∣X[g(Y)∣X=⋅]=CXYg. \mathcal C_{XX} \mathbb E_{Y|X}[g(Y)|X=\cdot] = \mathcal C_{XY}g. CXX​EY∣X​[g(Y)∣X=⋅]=CXY​g.

Further, for some x∈X\mathbf{x} \in \mathcal Xx∈X, the reproducing property states that

EY∣x[g(Y)∣X=x]=⟨EY∣X[g(Y)∣X],k(x,⋅)⟩H. \mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] = \langle \mathbb E_{Y|X}[g(Y)|X], k(\mathbf{x}, \cdot) \rangle_{\mathcal H}. EY∣x​[g(Y)∣X=x]=⟨EY∣X​[g(Y)∣X],k(x,⋅)⟩H​.

Combining the above the equations, and taking the conjugate transpose of CXX−1CXY\mathcal C_{XX}^{-1}\mathcal C_{XY}CXX−1​CXY​ gives

EY∣x[g(Y)∣X=x]=⟨g,CXYCXX−1k(x,⋅)⟩G=⟨g,UY∣x⟩G. \mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] = \langle g, \mathcal C_{XY}\mathcal C_{XX}^{-1}k(\mathbf{x},\cdot) \rangle_{\mathcal{G}} = \langle g, \mathcal U_{Y|\mathbf{x}} \rangle_{\mathcal{G}}. EY∣x​[g(Y)∣X=x]=⟨g,CXY​CXX−1​k(x,⋅)⟩G​=⟨g,UY∣x​⟩G​.

In an infinite RKHS, CXX−1\mathcal C_{XX}^{-1}CXX−1​ does not exist, so a regularised version is often used, that is [33] [28] :

UY∣X:=CXY(CXX+λI)−1, \mathcal U_{Y|X} := \mathcal C_{XY}(\mathcal C_{XX} + \lambda \mathcal I)^{-1}, UY∣X​:=CXY​(CXX​+λI)−1,

where λ>0\lambda > 0λ>0 is the regularization parameter and I\mathcal II is the identitiy operator in H\mathcal HH. Fukumizu et al. [34] stated that, under some mild conditions, the empirical estimator is a consistent estimator of EY∣x[g(Y)∣X=x]\mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}]EY∣x​[g(Y)∣X=x].

In practice, the joint distribution P(X,Y)\mathbb P(X,Y)P(X,Y) is unknown, and CXY\mathcal C_{XY}CXY​ and CXX\mathcal C_{XX}CXX​ cannot be calculated directly. To overcome this, consider the i.i.d. sample (x1,y1),...,(xn,yn)(\mathbf{x}_{1},\mathbf{y}_{1}),...,(\mathbf{x}_{n},\mathbf{y}_{n})(x1​,y1​),...,(xn​,yn​) from P(X,Y)\mathbb P(X,Y)P(X,Y). Let Y:=[ϕ(x1),...,ϕ(xn)]TY := [\phi(\mathbf{x}_{1}),...,\phi(\mathbf{x}_{n})]^{T}Y:=[ϕ(x1​),...,ϕ(xn​)]T and Φ:=[φ(y1),...,φ(yn)]T\Phi := [\varphi(\mathbf{y}_{1}),...,\varphi(\mathbf{y}_{n})]^{T}Φ:=[φ(y1​),...,φ(yn​)]T where ϕ:X→H\phi : \mathcal X \to \mathcal Hϕ:X→H and φ:Y→G\varphi : \mathcal Y \to \mathcal Gφ:Y→G are the feature maps corresponding to the respective kernels kkk and lll. The associated Gram matrices are K=YTYK = Y^{T}YK=YTY and L=ΦTΦL = \Phi^{T}\PhiL=ΦTΦ. With this information, the empirical estimator of the conditional mean embedding becomes [33]

C^XY(C^XX+λI)−1k(x,⋅)=1nΦYT(1nYYT+λI)−1k(x,⋅)=ΦYT(YYT+nλI)−1k(x,⋅)=Φ(YTY+nλIn)−1YTk(x,⋅)=Φ(K+nλIn)−1kx \hat{\mathcal C}_{XY}(\hat{\mathcal C}_{XX} + \lambda \mathcal I)^{-1}k(\mathbf{x},\cdot) = \frac{1}{n}\Phi Y^{T}(\frac{1}{n}YY^{T} + \lambda \mathcal I)^{-1}k(\mathbf{x},\cdot)\\ = \Phi Y^{T}(YY^{T} + n\lambda \mathcal I)^{-1}k(\mathbf{x},\cdot)\\ = \Phi (Y^{T}Y + n\lambda \mathbf{I}_n)^{-1}Y^{T}k(\mathbf{x},\cdot)\\ = \Phi(\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}} C^XY​(C^XX​+λI)−1k(x,⋅)=n1​ΦYT(n1​YYT+λI)−1k(x,⋅)=ΦYT(YYT+nλI)−1k(x,⋅)=Φ(YTY+nλIn​)−1YTk(x,⋅)=Φ(K+nλIn​)−1kx​

So, the conditional mean embedding of µY∣xµ_{Y|\mathbf{x}}µY∣x​ is estimated by [33]

µ^Y∣x=Φ(K+nλIn)−1kx. \hat{µ}_{Y|\mathbf{x}} = \Phi(\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}}. µ^Y∣x​=Φ(K+nλIn​)−1kx​.

Similar as the embedding of marginal distributions in equation 12, the embedding of conditional distributions can be written in terms of weights. Consider βλ^:=(K+nλIn)−1kx∈Rn\hat{\mathbf{ \beta}_{\lambda}} := (\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}} \in \mathbb R^{n}βλ​^​:=(K+nλIn​)−1kx​∈Rn. Subsequently, equation 22 can be written as µ^Y∣x=Φβλ^=∑i=1n(βλ^)iφ(yi)\hat{µ}_{Y|\mathbf{x}} = \Phi \hat{\mathbf{ \beta}_{\lambda}} = \sum_{i=1}^{n}(\hat{\mathbf{\beta}_{\lambda}})_i\varphi(\mathbf{y}_i)µ^Y∣x​=Φβλ​^​=∑i=1n​(βλ​^​)i​φ(yi​). It is important to note that in this case the weights β\mathbf{\beta}β depend on the value of the conditioning variable XXX instead of being uniform [33].

Learning on Distributional Data

The conditional mean embedding has a natural interpretation as a solution to a vector-valued regression problem, observed first by Zhang et al. [35] and later by Grünewälder et al. [36]. As discussed earlier, the conditional mean embedding is defined via EY∣x[g(Y)∣X=x]=⟨g,µ^Y∣x⟩G\mathbb E_{Y|\mathbf{x}}[g(Y)|X=\mathbf{x}] = \langle g, \hat{µ}_{Y|\mathbf{x}} \rangle_{\mathcal{G}}EY∣x​[g(Y)∣X=x]=⟨g,µ^Y∣x​⟩G​, i.e., for every x∈X\mathbf x \in \mathcal Xx∈X, µ^Y∣x\hat{µ}_{Y|\mathbf{x}}µ^Y∣x​ is a function on Y\mathcal YY and by that defines a mapping from X\mathcal XX to G\mathcal GG. Moreover, the empirical estimator µ^Y∣x=Φ(K+nλIn)−1kx\hat{µ}_{Y|\mathbf{x}} = \Phi(\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}}µ^Y∣x​=Φ(K+nλIn​)−1kx​, suggests that the conditional mean embedding is the solution to an underlying regression problem.

Consider the i.i.d. sample (x1,z1),...,(xn,zn)∈X×G(\mathbf{x}_{1},\mathbf{z}_{1}),...,(\mathbf{x}_{n},\mathbf{z}_{n}) \in \mathcal X \times \mathcal G(x1​,z1​),...,(xn​,zn​)∈X×G, a vector-valued regression problem can be formulated as:

ε^λ(f)=∑i=1n∥zi−f(xi)∥G2+λ∥f∥HΓ2, \hat{\varepsilon}_{\lambda}(f) = \displaystyle\sum_{i=1}^{n} {\|\mathbf{z}_{i} - f(\mathbf{x}_{i})\|}_{\mathcal G}^{2} + \lambda {\|f\|}_{\mathcal {H}_{\Gamma}}^{2}, ε^λ​(f)=i=1∑n​∥zi​−f(xi​)∥G2​+λ∥f∥HΓ​2​,

where G\mathcal GG is a Hilbert space, HΓ\mathcal {H}_{\Gamma}HΓ​ an RKHS of vector-valued functions from X\mathcal XX to G\mathcal GG, and ε^λ\hat{\varepsilon}_{\lambda}ε^λ​ is error of the associated regression problem [37]. Grünewälder et al. [36] states that by minimizing the optimization in equation 23, µ^Y∣x\hat{µ}_{Y|\mathbf{x}}µ^Y∣x​ can be obtained. So, the natural optimization problem for the conditional mean embedding is to find a function µ:X→Gµ : \mathcal X \to \mathcal Gµ:X→G that minimizes an objective. This objective can be bounded from above by a surrogate loss function, which can be described by its empirical counterpart [36] :

ε^S[µ]=∑i=1n∥l(yi,⋅)−µ(xi)∥G2+λ∥µ∥HΓ2. \hat{\varepsilon}_{\mathcal S}[µ] = \displaystyle\sum_{i=1}^{n} {\|l(\mathbf{y}_i,\cdot) - µ(\mathbf{x}_i)\|}_{\mathcal G}^{2} + \lambda {\|µ\|}_{\mathcal {H}_{\Gamma}}^{2}. ε^S​[µ]=i=1∑n​∥l(yi​,⋅)−µ(xi​)∥G2​+λ∥µ∥HΓ​2​.

The added regularization term provides a well-posed problem and prevents overfitting. Interpretating the conditional mean embedding as a solution to a vector-valued regression problem gives the advantage of being able to use cross-validation or model selection, due to the well-defined loss function. Since G\mathcal GG is assumed to be finite-dimensional, the conditional mean embedding is the ridge regression of the feature vectors. Consider βλ^:=(K+nλIn)−1kx\hat{\mathbf{ \beta}_{\lambda}} := (\mathbf{K} + n\lambda \mathbf{I}_{n})^{-1}\mathbf{k}_{\mathbf{x}}βλ​^​:=(K+nλIn​)−1kx​, in a ridge regression context the hat matrix Hλ\mathbf{H}_{\lambda}Hλ​ is:

Hλkx=k^x=Φβλ^Hλ=K(K+λI)−1. \mathbf{H}_{\lambda}\mathbf{k}_{\mathbf{x}} = \hat{\mathbf{k}}_{\mathbf{x}} = \Phi \hat{\mathbf{ \beta}_{\lambda}}\\ \mathbf{H}_{\lambda} = \mathbf{K}(\mathbf{K} + \lambda \mathbf{I})^{-1}. Hλ​kx​=k^x​=Φβλ​^​Hλ​=K(K+λI)−1.

Using leave-one-out cross-validation (LOOCV), the estimated conditional embedding is defined as [38] :

µ^Y∣xLOOCV=(I−diag(Hλ))−1(Hλ−diag(Hλ))Φ, \hat{µ}_{Y|\mathbf{x}}^{LOOCV} = (\mathbf{I} - diag(\mathbf{H}_{\lambda}))^{-1}(\mathbf{H}_{\lambda} - diag(\mathbf{H}_{\lambda}))\Phi, µ^Y∣xLOOCV​=(I−diag(Hλ​))−1(Hλ​−diag(Hλ​))Φ,

with diag(⋅)diag(\cdot)diag(⋅) the diagonal matrix. It's important to note that using the above equation, all LOOCV conditional embeddings can be calculated at once using matrix multiplications.

To interpretate the obtained results, the underlying distributions needs to be recovered from the embeddings, which is the topic of the next section.

Recovering Distributions from RKHS Embeddings

Recovering information of P\mathbb PP from the kernel mean embedding µPµ_{\mathbb P}µP​ is known as the distributional pre-image problem [39] [40]. In this context, objects in the input space which correspond with a specific kernel mean embedding in a feature space, are looked for. Consider Pθ\mathbb{P}_{\theta}Pθ​ an arbitrary distribution parametrized by θ\mathbf{\theta}θ and its mean embedding in H\mathcal HH, µPθµ_{\mathbb{P}_{\theta}}µPθ​​. By solving following minimization problem Pθ\mathbb{P}_{\theta}Pθ​ can be found:

θ^=arg min⁡θ∈Θ∥µ^Y−µPθ∥H2=arg min⁡θ∈Θ⟨µ^Y,µ^Y⟩−2⟨µ^Y,µPθ⟩+⟨µPθ,µPθ⟩, \hat{\mathbf{\theta}} = \argmin_{\theta \in \Theta}{\|\hat{µ}_{Y} - µ_{\mathbb{P}_{\theta}}\|}_{\mathcal H}^{2}\\ = \argmin_{\theta \in \Theta} \langle \hat{µ}_{Y},\hat{µ}_{Y} \rangle -2\langle \hat{µ}_{Y}, µ_{\mathbb{P}_{\theta}} \rangle + \langle µ_{\mathbb{P}_{\theta}}, µ_{\mathbb{P}_{\theta}} \rangle, θ^=θ∈Θargmin​∥µ^Y​−µPθ​​∥H2​=θ∈Θargmin​⟨µ^Y​,µ^Y​⟩−2⟨µ^Y​,µPθ​​⟩+⟨µPθ​​,µPθ​​⟩,

where θ\thetaθ is subject to appropriate constraints. As seen earlier, equation 27 describes the MMD. ⟨µ^Y,µ^Y⟩\langle \hat{µ}_{Y},\hat{µ}_{Y} \rangle⟨µ^Y​,µ^Y​⟩ is constant and is left out of the minimization. Assume µPθ=∑i=1nαiφ(yi)µ_{\mathbb{P}_{\theta}} = \sum_{i=1}^{n}{\alpha}_i\varphi(\mathbf{y}_i)µPθ​​=∑i=1n​αi​φ(yi​) for some α∈Δn−1\alpha \in \Delta^{n-1}α∈Δn−1, i.e., Pθ\mathbb{P}_{\theta}Pθ​ is a histogram.

w=[wi]∈Δn−1\mathbf w = [w_i] \in \Delta^{n-1}w=[wi​]∈Δn−1, i.e., a histogram with weights subject to the constraint ∑inwi=1\sum_{i}^{n}w_i = 1∑in​wi​=1 and wi>0w_i > 0wi​>0 [31].
